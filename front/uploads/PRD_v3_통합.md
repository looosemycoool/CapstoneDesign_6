## 1. 문서 개요

### 1-1. 문서 목적

본 문서는 **KNU 공지사항 챗봇 — 하이브리드 RAG 지능형 캠퍼스 가이드** 서비스의 기능 요구사항, 화면 설계, 기술 명세를 정의한다. PRD v2.0과 팀 기능명세서 PDF를 통합하여 두 문서 간 충돌·누락 항목을 해소한 단일 진실 원천(Single Source of Truth)이다.

### 1-2. 범위

- **In Scope**: 프론트엔드(React/TS), 백엔드 API 인터페이스, 공지사항 크롤링 연동
- **Out of Scope**: 크롤러 자체 구현 세부 사항, 인증/로그인, 관리자 페이지

### 1-3. 이해관계자

| 역할 | 이름/조직 | 관심사 |
|---|---|---|
| 제품 오너 | 6조 전체 | 기능 완성도, 일정 |
| 개발 (FE) | 프론트엔드 담당 | API 스펙, 컴포넌트 설계 |
| 개발 (BE) | 백엔드 담당 | DB 스키마, 크롤링 파이프라인 |
| 최종 사용자 | 재학생 | 빠른 공지 조회, 편의성 |

## 2. 배경 및 문제 정의

### 2-1. 현황 분석

학교 포털의 공지사항 시스템은 다음과 같은 구조적 문제를 가지고 있다.

```
현재 공지사항 탐색 경로
학생 → 포털 접속 → 카테고리 선택 → 게시판 목록 → 페이지 이동 → 원문 확인
                     ↑
           (학사, 장학, 취업, 행사 등
            10개+ 카테고리로 분산)
```

| 문제 | 영향 |
|---|---|
| 카테고리가 분산되어 어떤 게시판에 공지가 있는지 모름 | 탐색 시간 증가 (평균 5분+) |
| 포털 내 검색 기능이 키워드 일치에만 의존 | 자연어 질의 불가 |
| 캠퍼스 은어·줄임말로 검색 시 결과 품질 저하 | 검색 실패율 증가 |
| 모바일 환경에서 포털 UX가 불편 | 접근성 저하 |
| 오래된 공지와 최신 공지가 혼재 | 정보 신뢰도 저하 |

### 2-2. 솔루션 개요

```
제안 시스템
학생 → 챗봇 입력 → [줄임말 치환 파이프라인]
                          ↓
                    [하이브리드 RAG 엔진]
                          ↓
               자연어 응답 + 출처 카드
                          ↑
              크롤링된 공지 DB (매일 새벽 2시 갱신)
              + LLM 자연어 이해
```

### 3-2. 사용 맥락 (Context of Use)

| 맥락 | 세부 상황 | 주요 니즈 |
|---|---|---|
| 수업 전 이동 중 | 지하철/버스, 모바일 | 빠른 답변, 간결한 UI |
| 수강신청 기간 | PC, 여러 탭 열기 | 정확한 일정 정보 |
| 장학금 시즌 | 모바일/PC | 신청 방법·마감일, 즐겨찾기 저장 |
| 졸업 직전 | PC | 졸업 요건 상세 확인, 출처 원문 링크 |

```ts
  isStreaming: boolean;
  abortController: AbortController | null;
  lastUserMessage: string | null;    // 에러 시 재시도용 직전 메시지 보존

  setSessionId: (id: string | null) => void;
  addMessage: (message: Message) => void;
  appendStreamChunk: (chunk: string) => void;
  finalizeStream: (sources?: AnnouncementSource[]) => void;
  setStreaming: (value: boolean) => void;
  setAbortController: (ctrl: AbortController | null) => void;
  setLastUserMessage: (msg: string) => void;
  resetChat: () => void;
  loadSession: (messages: Message[], sessionId: string) => void;
}
```

**상태 전이 다이어그램**
```
[초기] sessionId=null, messages=[], isStreaming=false
   │
   │ addMessage(user) + setLastUserMessage
   ▼
[전송] messages=[userMsg], isStreaming=true
   │
   │ appendStreamChunk (반복)
   ▼
[스트리밍] messages=[userMsg, {role:assistant, content:"..."}]
   │
   ├── finalizeStream → [완료] isStreaming=false, sources 추가
   ├── abort()        → [중단] isStreaming=false, 현재 텍스트 유지
   └── error          → [에러] isStreaming=false, isError=true 메시지
   │
   │ resetChat
   ▼
[초기]
```

## 16. 용어 정의

| 용어 | 정의 |
|---|---|
| 세션 (Session) | 사용자의 하나의 연속된 대화 단위 |
| 스트리밍 (Streaming) | SSE를 통해 응답 텍스트를 청크 단위로 실시간 전송하는 방식 |
| 청크 (Chunk) | 스트리밍 응답의 단위 데이터 |
| 아코디언 | 클릭 시 내용이 접히고 펼쳐지는 UI 컴포넌트 |
| 출처 카드 | 챗봇 응답이 참조한 공지사항을 시각적으로 표시하는 카드 UI |
| 하이브리드 | RAG(검색 기반) + LLM 생성의 결합 방식 |
| RAG | Retrieval-Augmented Generation — 벡터 검색으로 관련 문서를 찾아 LLM에 컨텍스트로 제공 |
| 멀티턴 | 이전 대화 맥락을 유지하며 이어지는 다회차 대화 |
| 웰컴 화면 | 대화가 없을 때 첫 진입 시 표시되는 안내 및 추천 질문 화면 |
| TTFT | Time To First Token — 메시지 전송 후 첫 번째 텍스트 청크를 수신하기까지의 시간 |
| 줄임말 치환 | 캠퍼스 은어를 공식 명칭으로 변환하는 FE 파이프라인 처리 |
| 즐겨찾기 (★) | 특정 어시스턴트 답변을 내 보관함에 영속 저장하는 기능 |
| 내 보관함 | 즐겨찾기된 답변 목록을 열람하는 사이드바 탭 |
| 콜드 스타트 (Cold Start) | 사용자가 처음 접속하여 어떤 질문을 해야 할지 모르는 초기 상태 |
| Hydration | 서버 데이터를 클라이언트 상태(Context)로 완전히 로드하는 과정 |
