## ② 파서 예외조건 어노테이션 (`pipeline/01_parser.py`)

### 배경

조건 누락 오답 11개 중 일부는 예외 조항(`※`, `단,`, `다만,`, `~이내`, `~에 한함`)이
주 규정과 **다른 청크**로 분리되어 벡터 유사도 검색에서 아예 후보에 오르지 못하는 케이스.
`_annotate_track_sections()`가 트랙 구분을 해결했듯, 예외 조항도 명시적 태깅으로
임베딩 공간에서 주 규정과 가까워지도록 유도.

### 구현 방향

```python
def _annotate_exception_clauses(text: str) -> str:
    """
    예외·조건 패턴을 감지해 [조건], [예외], [유효기간] 태그 삽입.

    패턴 예시:
      ※ 졸업사정자료 제출일로부터 2년 이내 성적만 유효
      단, 편입생은 조기졸업 불가
      다만, 현장실습 미이수 시 창업교과목 +6학점 추가

    변환 후:
      [유효기간] 졸업사정자료 제출일로부터 2년 이내 성적만 유효
      [예외] 편입생은 조기졸업 불가
      [조건] 현장실습 미이수 시 창업교과목 +6학점 추가
    """
```

`parse_pdf_batch()` 내 `_annotate_markdown_table_rows()` 다음 단계로 추가.

### 적용 시 필요 작업

1. `pipeline/01_parser.py` 함수 추가 및 `parse_pdf_batch()` 호출 체인 연결
2. 전체 재파싱: `python pipeline/01_parser.py`
3. Vector DB 재구축: `python pipeline/02_vector_db.py`
4. Graph DB 재구축: `python pipeline/03_graph_db.py --rebuild`

### 예상 효과

- 유효기간 관련 오답 (P1Q4 토익 유효기간, P8Q5 1년 이내 조건 등) 개선
- 예외 대상 관련 오답 (P4Q4 편입생 조기졸업 불가 등) 개선

## ④ Reranker 예외조건 키워드 가중치 (`pipeline/04_hybrid_rag.py`)

### 배경

① 인접 청크 확장 적용 이후에도, 예외 조항을 담은 청크가 Reranker에서 상위에 올라오지
못해 `RERANK_TO_K` 컷에서 잘리는 케이스가 남을 경우 적용.
현재 `llm_rerank()` 프롬프트는 주제 관련성만 판단하고, 예외·조건 포함 여부는 명시하지 않음.

### 구현 방향

`llm_rerank()` 프롬프트에 아래 지시 추가:

```
예외 조건(※, 단, 다만, ~이내, ~이상, ~이하, ~에 한함, ~은 제외)이 포함된 청크는
질문의 조건 해석과 관련 있다면 일반 설명 청크보다 높은 점수를 부여하세요.
```

### 적용 조건

① 인접 청크 확장 적용 후 재평가에서 여전히 Reranker 컷에서 예외청크가 밀리는 패턴이
확인될 때 추가. ①과 중복 효과가 있어 ① 단독으로 충분할 수 있음.

### 적용 시 필요 작업

1. `pipeline/04_hybrid_rag.py` 내 `llm_rerank()` 프롬프트 수정
2. 재평가로 효과 검증
